# Documentation Standards

When maintaining the `README.md` or template documentation:

1. **User-Centric**: Documentation must answer the "How do I start?" question within the first two sections.
2. **Code Examples**: Provide copy-pasteable blocks for common tasks (e.g., bootstrapping, environment variable setup).
3. **Template Context**: If documenting a template, clearly state the prerequisites on the *host machine* vs. the *container*.
4. **Consistency**: Ensure all code snippets in READMEs use the directory structures we have agreed upon (`/workspaces/workspace`, `.devcontainer/`).
